2. VERIFICAR HEADERS:
   DEBE incluir en TODAS las peticiones:
   headers: {
     'empresaId': '1'
   }

3. VERIFICAR QUERY PARAMS:
   DEBE incluir empresaId=1 en la URL:
   ?empresaId=1

4. VERIFICAR METODO HTTP:

=== TROUBLESHOOTING ===

PROBLEMA: Frontend no recibe datos
SOLUCION: 
1. Verificar que backend esté corriendo: http://localhost:8080
2. Probar endpoint en navegador directamente
3. Revisar consola del navegador (F12) para ver errores
4. Verificar que se incluye header empresaId
5. Verificar que se incluye query param ?empresaId=1

PROBLEMA: Error 400 Bad Request
SOLUCION:
- Falta header empresaId: agregar headers: {'empresaId': '1'}
- Falta query param: agregar ?empresaId=1 a la URL

PROBLEMA: Error 404 Not Found
SOLUCION:
- URL incorrecta, verificar que sea: /api/obras/todas?empresaId=1
- Backend no está corriendo en puerto 8080

PROBLEMA: Frontend muestra 0 obras pero backend devuelve 9
SOLUCION:
- El frontend está consultando un endpoint diferente
- Buscar en el código del frontend por "obras" o "api/obras"
- Verificar que use el endpoint correcto: /api/obras/todas

PROBLEMA: CORS error
SOLUCION:
- El backend ya tiene CORS configurado para http://localhost:4200
- Si el frontend corre en otro puerto, avisar para agregarlo
